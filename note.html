1. mysql -u 계정명 -p (기본 로그인)
2. mysql -u 계정명 -h 서버주소 -p (다른 서버로 접속)
3. mysql -u 계정명 -h 서버주소 -p 포트번호 -p (다른 서버에 다른 포트로 접속)

dir(Directory lists) (enter)

host : 접속주소(접속가능한 주소-접속하도록 허용된 주소),

사용자 생성

2. create user '계정명'@'접속주소';
3. create user '계정명'@'접속주소'identified by '비밀번호';

문자셋
컴퓨터에서 문자를 표시하기 위한 코드표(아스키코드표)
현재는 전부 유니코드로 통합되어 utf-8 방식이 표준으로 사용되고 있으나 예전 버전에는 문자셋이 기본이었기 때문에 utf-8 로 설정이 필요했음.

utf-8 : 가변형 3byte
utf-16 : 기본, 고정형 4byte, 모든 문자를 표현가능
utf-32 : 이모티콘 등까지 전부 가능)


database 추가
create database 데이터베이스명; ('문자' 형태로 적어도 구동가능)
create database 데이터베이스명 default character set utf8; (현재 버전은 상관없지만, 유지보수를 위해 알아둬야함.)


database 삭제
drop database 데이터베이스명;


database 사용권한
1. 권한 확인 : show grants for '계정명'@'접속주소';
2. 권한 설정 : grant all privileges on 데이터베이스명.테이블명(* : 모든 테이블의 권한을 주는 특수문자) to '계정명'@'접속주소';
3. 권한 삭제 : revoke all on 데이터베이스명.테이블명 from '계정명'@'접속주소';

권한을 설정할 때, 데이터베이스명에 오타가 나면 오타가난대로 데이터베이스 권한이 주어지기 때문에, 데이터베이스명의 오타를 주의해야한다.
