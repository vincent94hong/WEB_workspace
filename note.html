SQL 명령어 구분
1. DDL : 데이터 정의어, 데이터 베이스 관리자가 사용, 데이터 베이스를 설꼐 및 생성하는 언어. create / drop / alter
2. DML : 데이터 조작어, 일반 사용자/프로그램 개발자가 사용, 데이터 베이스에 실제 데이터를 추가, 삭제, 수정, 조회하기 위해 사용하는 언어. select / insert / update / delete
3. DCL : 데이터 관리어, 데이터 베이스 관리자가 사용, 데이터 베이스 사용 권한 및 사용자 권한 제어하기 위해 사용하는 언어. grant / revoke

위에서 사용된 언어들이 1, 3번 DDL, DCL 인 것.
앞으로 DML 을 주로 사용하게 될 것.


SQL 명령어 상세
1. Select : 데이터를 조회하기 위한 명령어, where 구문을 함께 사용하여 조회시 필요한 조건을 추가하여 데이터를 조회함.
-사용법 1 : select 컬럼명1, 컬럼명2, ... from 테이블명;
-사용법 2 : select 컬럼명1, 컬럼명2, ... from 테이블명 where 조건;
* = all

2. Distinct : select 구문의 컬럼명과 함께 사용하는 명령어로 중복되는 데이터를 제거
-사용법 : select distinct 컬럼명 ... from 테이블명;

3. Where : 데이터베이스에서 데이터를 조회할 때, 조건을 추가하는 명령어이며, 조건에 만족하는 데이터만 출력함.
select / update / delete 에서 사용됨
-사용법 1 : select 컬럼명 ... from 테이블명 where 검색조건;
ex) select * from employees where first_name = 'mario'; > 성이 마리오인 사람을 조회/검색
-사용법 2 : update 테이블명 set 컬럼명 ... where 검색조건;
-사용법 3 : delete from 테이블명 where 검색조건;

4. And / Or : where 구문을 사용하여 검색한 조건에 추가 조건을 사용하고자 할 경우 사용. where 구문에 추가하여 사용.
-And : 조건이 추가될 때마다 검색 정확도가 올라감
-Or : 조건이 추가될 때마다 검색되는 데이터의 양이 늘어남
-사용법 : select 컬럼명 ... from 테이블명 where 조건1 and(or) 조건2, and(or) 조건3, ...

5. Not : 검색 조건에 부합하지 않는 데이터를 출력할때 사용. -사용법 : select 컬럼명 ... from 테이블명 where not 검색조건; ex) select * from employees where not gender = 'M';
ex) select * from employees where not gender = 'M' and not first_name = 'mario';

6. Order by : 지정한 컬럼을 기준으로 데이터를 정렬하여 출력, ASC(오름차순), DESC(내림차순), where 구문 마지막에 입력. 여러 정렬 조건을 할 수 있으나 세개이상 지정시 원하는 대로 정렬되지 않을 수 있음.
-사용법 : select 컬럼명 ... from 테이블명 where 검색조건 order by 컬럼명 정렬방식(asc / desc);
ex) select * from employees order by first_name asc;
ex) select * from employees where emp_no < 15000 order by first_name asc;
ex) select * from employees order by birth_date asc, first_name asc, last_name desc;

7. Insert : 데이터 베이스에 데이터를 추가하는 명령어
데이터 입력시 칼럼의 순서는 상관없음(사용자가 입력한 칼럼 순서에 맞는 데이터만 입력하면 됨), AI/Default로 설정된 데이터는 입력시 컬럼을 생략해도 됨, NotNull로 설정된 부분만 정확하게 데이터를 입력하면 됨, Unique로 설정된 키는 중복되는 데이터를 입력하면 안됨.
-사용법 1 : insert into 테이블명 (칼럼명, ... ) values (데이터1 ...);
-사용법 2 : insert into 테이블명 (칼럼명, ... ) values (데이터1-1, ...), (데이터2-1, ... ), ...;

2. 우클릭, create table.
3. table name 설정 후, column name 추가

*PK : 기본키 - 테이블 내에서 데이터를 구별하기 위한 유일한 값.
*NN(Not Null)(Null-데이터가 없음을 뜻함) : 데이터가 비어있으면 안됨.
*AI(Auto Increment) : 자동 값 증가 (INT 타입-숫자-만 지원)
*UQ(Unique) : 유일한 값을 의미. 중복데이터 허용 안함.
-PK를 키면 NN & UQ 가 자동으로 켜짐. UQ는 PK가 아니면서도 유일한 값이 필요할 때 사용.
-ex) 회원번호는 PK, 회원아이디는 UQ를 사용

데이터 베이스에 저장되어 있는 데이터를 수정하는 명령어
주의사항 : 조건식 부분(where)을 생략하게 되면 해당 테이블의 모든 데이터를 수정
사용법 : update 테이블명 set 컬럼명1 = 데이터1, 컬럼명2 = 데이터2, ... where 조건식

DELETE
데이터 베이스에 저장되어 있는 데이터를 삭제하는 명령어
주의사항 : 조건식을 생략할 경우 해당 테이블의 모든 데이터를 삭제
사용법 : delete from 테이블명 where 조건식;

LIMIT (MySql 에서만 사용됨.)
MySql 에서 출력하는 데이터를 제한하는 명령어, Where 절에서 사용되는 명령어
사용법1 : ~ where 조건식 limit 출력수(결과 갯수)
사용법2 : ~ where limit 출력번호, 출력수

MIN()/MAX()
지정한 컬럼의 최솟값/최댓값을 출력하는 함수, Select 절에서 사용
사용법 : select min(컬럼명) from 테이블명

COUNT()
지정된 기준과 일치하는 데이터 수를 출력, select 절에서 사용
사용법 : select count(컬럼명) from 테이블명

AVG()
지정된 컬럼(숫자일 경우에만)의 평균값을 출력, select 절에서 사용
사용법 : select AVG(컬럼명) from 테이블명

SUM()
지정된 컬럼(숫자일 경우에만)의 총 합계, select 절에서 사용
사용법 : select SUM(컬럼명) from 테이블명

LIKE
검색 패턴을 생성하는 명령어, 와일드카드문자와 함께 사용하여 패턴 생성, where절에서 사용
사용법 : ~ where 컬럼명 like '검색패턴'

WILD CARD
문자열에서 하나 이상의 문자를 대체하는 문자, _, % 가 존재
_ : 1개의 임의의 문자로 대체됨
% : 0개 이상의 임의의 문자로 대체됨

IN
검색에 대한 여러개의 데이터를 지정할 수 있는 명령어
서브쿼리 사용을 위한 명령어(고급기술/어려움), where 절에서 사용
or 명령어를 사용한 것과 동일한 효과
주의사항 : in/or 결과는 동일하지만, in 사용시 like 사용할 수 없고 or 사용시 like 사용 가능
사용법1 : ~ where 컬럼명 in (데이터1, 데이터2, ...)
사용법2 : ~ where 컬럼명 in (select ... -서브쿼리)

BETWEEN
지정한 범위 내의 데이터를 출력할 때 사용, where 절에서 사용
비교연산자와 and 연산자를 사용하여 between 과 동일한 결과를 얻을 수 있음
특징 : 범위 값은 숫자, 문자, 날짜로 사용 가능
주의사항 : 시작값과 종료값 포함
사용법 : ~ where 컬럼명 between 시작값 and 종료값

AS
테이블 및 컬럼명에 별칭을 부여하는 명령어, 테이블과 select 에서 사용
사용이유 : 컬럼명의 길이가 길어 수정하고자 할 경우, 화면에 출력하고자 하는 컬럼명을 변경할 경우, Join 사용시 테이블 명을 변경하고자 할 경우
주의사항 : 해당 쿼리문에서만 효과가 있음
사용법1 : select 컬럼명1 as '별칭1', 컬럼명2 as '별칭2', ... from 테이블명
사용법2 : select 별칭테이블명.컬럼명1, 별칭테이블명.컬럼명2, .. from 테이블명 as '별칭'
